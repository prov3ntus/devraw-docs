Sound Aliases & The SZC

This page details the SZC, sound aliases, what they are & how they're used.

Beginner

What is a Sound Alias?​

A "Sound Alias" is just a term used when talking about a sound within the game. Not just it's file path, but other properties as well.

The game needs to store the properties of each sound somewhere, so that the sound engine can play it how we want it to.

Some examples of "properties" that need to be stored:

  • FileSpec: The path to the actual sound file, relative to <BO3 Root>\sound_assets\.
  • Storage: Whether it's loaded into memory when the map loads, or streamed from disk.
  • PanType: If it's played in 3D space, or it's just a 2D sound.
  • DistMin / DistMax: Distance players can hear the sound from in-game (if PanType is 3D).

The list goes on. It's actually a big list, but you only need to be familiar with a few of them.

We want to be able to store and edit some of these properties easily. Call of Duty's solution is to use CSV files to store info for each sound.

CSV files store data in a table, which is perfect for sound info:

NameStorageFileSpecPanTypeDistMinDistMaxDry...
amb_alarmloadedzmb\...\alarm\alarm_00.wav3d5002500...
zmb_cha_chingloadedzmb\purchases\common\accept.wav3d250350...
zmb_laugh_childloadedzmb\purchases\box\laugh\child.wav2d010000...

The columns are all the sound properties you can set, and then each row is just what those properties are for each sound we create. Once you tell your map to load this CSV of sound aliases, you'll be able to access all these sounds across the engine through it's alias name (e.g. zmb_cha_ching, which plays every time you buy a door).

Sound aliases are commonly referenced in scripts (using the PlaySound() function), or set in APE to play at specific times in animations (e.g. playing a "mag out" sound on the frame that the player pulls the mag out).

Aliases with the same names

If multiple sound aliases have the same Name value, the sound engine will just pick a random one from between them every time it's played.

Loading Aliases Into Your Map​

As mentioned earlier, sound aliases are stored in CSV files, but the CSV files themselves are found here:

<BO3 Root>\share\raw\sound\aliases\*

When you first install the tools, the only file you'll have in this folder is .\user_aliases.csv.

The user aliases file is loaded into every map by default.

However it's not recommended to use it because when you want to compile different maps, you'll either have to comment out the sounds or swap them out. Also, user aliases tends to randomly stop compiling sounds correctly, and cause other sound issues. It might be some sort of engine issue.

At that point, you should just duplicate user aliases and use your own CSVs.

The Sound Zone Config​

If you want to load your own CSV, you have to update the Sound Zone Config file (the SZC).

In the launcher, right click on your map and press "Open Map Folder". Navigate to sound\zoneconfig\ and open <MapName>.szc. You'll see the user aliases entry, which is there by default.

Copy the user aliases entry and rename the "Name" : "user_aliases", and "Filename" : "user_aliases.csv", to match your custom CSV.



Do that for every CSV file you want to load into that map.

warning

Make sure you're copying everything properly as shown in the video above. Fullscreen it if you have to. Maintain the syntax of the SZC file - don't miss a comma, make sure you're copying the correct set of brackets, etc.

Editing Aliases & CSVs​

Duplicate user_aliases.csv from <BO3 Root>\share\raw\sound\aliases\* and rename it to your use case. We need the first line from it, because that defines the column names of the table.

The sound converter expects those exact column names, so don't edit the first line.

You can modify the second line that defines the test_sound alias, putting your own values in, and duplicate the line when you want to create a new alias.

Organisation

Sound Alias Properties​

There are loads of properties, and most of them are already documented by 3arc for us in <BO3 Root>\docs_modtools\Sound_Mod_Docs.pdf, but I've outlined the main ones you should be familiar with below. Also, some of the explanations in that PDF are confusing as hell tbh.

Sound PropertyRequirementDescriptionDefault ValueAccepted Values
NameRequiredThis is the name that you will reference when you need to play the sound.
FileSpecRequiredThe path to the .wav file, relative to <BO3Root>\sound_assets\. If my sound is at <BO3Root>\sound_assets\_pv\my_sound.wav, I'd put _pv\my_sound.wav in the FileSpec column, for example.
StorageOptionalLoad into RAM when the map loads, or get it from disk. You don't want to stream big sounds (like music / ee songs).loadedloaded, streamed
TemplateOptionalThe alias's properties will default to the ones defined in this template alias instead of the engine's defaults. See Alias Templates below for more details.Any templates found in <Root>\share\raw\sound\templates\*
BusOptional...

Again, 3arc documented a lot more of these in <BO3 Root>\docs_modtools\Sound_Mod_Docs.pdf, so go check that out if you can't find one here.

SOUND LIMITS

There is a hard sound limit of around 300 MB for total sound size in a map or mod. The game will crash upon launch if you are over this limit.

Alias Templates​

Templates are used to set default properties for a sound alias, so we can leave columns blank and it will fall back to the value set in the template.

It's a neat way to give loads of sounds shared properties that you can override per alias.

It also makes it easier to quickly add sounds which need very specific properties into your map. For example, Treyarch made templates for ambient sounds, explosions, physics impacts, pack-a-punched weapons, etc.

As an example, Skye made some templates for sounds in his weapon ports, as most of the sounds shared the same properties.

Of course, credits to Skye for the snippet of his templates.

Each line is a template. Look at the "Name" column: wpn_t9_shot_plr, wpn_t9_shot_loop_plr, wpn_t9_pap_shot_plr, etc. These are the names of the templates. You'd put a template name in the "Template" column of a sound alias, and it will use these properties as defaults.

UIN_MOD is the template used for the test_sound in user aliases, and that specifies some nice defaults for you. If you used UIN_MOD for a sound alias, you'd only need to edit 2 properties: Name and FileSpec.

Any other properties are taken care of by the template. You only fill them out if you want to. This makes it really convenient when registering lots of sound aliases of similar types.
