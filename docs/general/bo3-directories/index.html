BO3 Directories

Beginner

This section focuses on Black Ops III's directory system, and how dragging & dropping works.

It also covers which folders the compiler prioritises when building your map.

About directoriesâ€‹

When you installed Mod Tools, you'll have noticed more folders in your BO3 root. Some of them are: map_source, model_export, share, sound_assets and texture_assets. These are some of the main folders that store assets for the game. You'll be using them a lot, and you'll naturally get familiar with them. Sometimes specific files must be inside their respective folder for the game to read them.

For example, consider the directory above. I want to put this sound in the game. Sound files must be within sound_assets as the linker is searching <path to root>\Call of Duty Black Ops III\sound_assets\ PLUS whatever the file path for the sound is. In the case above, the full path would be:

"<path to root>\Call of Duty Black Ops III\sound_assets\" PLUS "pv\motd_announcer_finn\max_ammo.wav"

This is just because the devs wanted all sounds to be located inside the sound_assets folder. However this isn't the case for all files used by the game. For example, .xmodel_bin files and textures (PNGs & TIFs) can be put anywhere inside the Black Ops III folder or all its subfolders. But I still recommend to keep them in their respective folders. Files can get very messy very quickly, and could cause you greif later on if there's a lack of organisation, which is why in game dev you'll never find a company that doesn't have a logical data structure for their game files.

I recommend you make a custom folder of your username (or an abbreviation of it) in the main folders when you add assets to them, so you know which stuff is yours.

Dragging & Dropping / Installing Asset Packsâ€‹

Dragging & dropping is an easy way to install someone else's assets they've released. It's a standard format for sharing assets within the Mod Tools community. The person packages up all the assets by putting them all into the same folders they were originally in. So when you install it, you just drag the main folders (that have their files in) into your BO3 root, and the files will be in the same place as the person who shared the files.

The file paths will all be the same, so if I'm releasing that Mob of the Dead Announcer pack, I would put it in the same directory tree. Then when the game compiles your map, it will find the files in the same location (from the BO3 root folder)

Usually people leave instructions on how to install the asset pack, in case there are a few more steps than a simple drag 'n drop.

File overridesâ€‹

Just like sound_assets is a base for sound files, there is another 'base' folder used for common shared files, used by all maps, located in <BO3 root>\share\raw\**. Now there are a ton of folders here, but don't let that intimiadte you. For now, let's just focus on the scripts folder.

This is, as the name implies, where all the global scripts are stored. What do I mean by global? Well, these scripts can be called and used in any map (assuming you've zoned them, of course). This also applies to every other file in the share\raw\** directory.

But what if you wanted to change a script, or another file for a specific map? If you change it in share\raw, it'll reflect the changes whenever you compile any map. Well, before the linker checks if the file is in share\raw, it'll actually check if it's in your map folder first.
