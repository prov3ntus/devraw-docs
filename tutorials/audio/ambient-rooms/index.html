Ambient Rooms

Everything about ambient rooms, with a walkthrough of setting up reverb effects for different rooms in your maps.

Intermediate

Before starting this tutorial, make sure you know:

  • How to set up a sound alias

Creating an Ambient File​

Navigate to <BO3 ROOT>\share\raw\sound\ambients and duplicate ambient_mod.csv. Rename it to something you will remember.

Organisation

It's smart to stay organized. For this, I would name the ambient file zm_<MapName>_ambient.csv.

Open the newly created ambient file in Microsoft Excel, LibreOffice or a similar program/app that can read CSV files. It is always much easier to understand/read if you view it in a spreadsheet viewer.

Let's now go over every important column that you may want to edit in this file. If it's not listed here, don't edit it (unless you know what you're doing).

Column NameDescriptionExample Value
NameThis is the name of the ambient room that we will reference later in Radiant.outdoor_alley
DefaultRoomUse this as the default ambient room across the whole map (where there is no ambient room trigger). You must have exactly one row that has this set to yes.yes or no
ReverbWhat reverb type to use (see "Stock Reverb List" below)global_hallway
ReverbDryLevelOn a scale from 0 to 1, how loud is the DRY sound? 0 --> 10.75
ReverbWetLevelOn a scale from 0 to 1, how loud is the WET sound? 0 --> 10.25
EntityContextValue0Is this room indoors or outdoors?indoor or outdoor
EntityContextValue1Is this room over or under water?over or under
LoopA sound alias to loop while inside the trigger. Useful for wind on a balcony, underwater ambience, etc.evt_plr_fire_loop
DuckWhat duck to use (see "Stock Ducks List" below). This tells the sound engine which sounds take priority, which ones are "ducked" (volume is lowered), if they're muffled, etc.cmn_swimming
note

"Mandatory" means the column must contain a value!

Setting up the Sound Zone Config​

Open your map's SZC file by going to <BO3 Root>\usermaps\zm_<MapName>\sound\zoneconfig\zm_<MapName>.szc.

There are a few changes we need to make to get ambient rooms set up correctly.

Adding the Map File Path​

First, lets enter the name of your map file into "MapFile" : "", (line 12 of the SZC). The start of the file should look like this:

<BO3 Root>\usermaps\zm_<MapName>\sound\zoneconfig\zm_<MapName>.szc
{
    "Name" : "zm_<MapName>",
    "GameMode" : "mpl",
    "IsCommon" : false,
    "Parent" : "",
    "Overlay" : "",
    "IsStandalone" : true,
    "IsProduction" : false,
    "IsShipped" : false,
    "DontDeploy" : false,
    "NoStreamBank" : false,
    "MapFile" : "",
    "Standalone" : true,
    "Builds" : [ "T7" ],
    ...
}

By default, your map file path isn't filled out. So you'll need to add it yourself, like so:

"MapFile" : "zm\\zm_<MapName>.map",

"MapFile" is expected to be a path to the .map file that Radiant saves to, relative to <BO3 Root>\map_source\, which is why we need a zm\\ at the start of the string there.

warning

It's important you add two backslashes for the map file path here, not one.

Replacing the Stock Ambient Room​

Next, let's replace the stock ambient room CSV with the one we made.

<BO3 Root>\usermaps\zm_<MapName>\sound\zoneconfig\zm_<MapName>.szc
{
    "Type" : "AMBIENT",
    "Name" : "ambient_mod",
    "Filename" : "ambient_mod.csv",
    "Specs" : [
        "mpl_mod"
    ]
},

Replace ambient_mod in both highlighted lines with whatever you named the file. If you listened to our naming recommendation, it'll be called zm_<MapName>_ambient.csv.

"Name" : "zm_<MapName>_ambient",
"Filename" : "zm_<MapName>_ambient.csv",

Radiant Setup​

Last but not least, we're going to define which areas use the ambient room(s) we just defined in the ambient CSV.

In Radiant, open the Entity Browser by pressing B and place a trigger_multiple in your map. With the trigger selected, open the Texture Browser by pressing T and search for sound_trigger and assign the texture to our new trigger.

Let's add some KVP's to the trigger:

KeyValue
CLIENTSIDE_TRIGGERtrue
targetnameambient_package
script_ambientroomThis is the name of the ambient room we put in the Name column of the ambient room CSV. The example name used earlier was outdoor_alley.
script_ambientpriority0 -> 16 with the higher number taking priority of other ambient rooms if triggers overlap each other

You may now manipulate this trigger however you need to fit your spaces, as well as copy/paste as many as you like. Triggers with different script_ambientroom KVPs can overlap, whichever one has a higher number in the script_ambientpriority will play over the others.

If you want to save on entities and/or you have a very complex area of your map that requires lots of triggers, you can make multiple brushes one entity:

  • Draw as many brushes as you need & select them all
  • From the Entity Browser, drag a trigger_multiple onto the camera/grid view
  • Make sure to still give it the sound_trigger texture from the Texture Browser

Each brush will be treated as the same entity, and hence share KVPs.

warning

Ambient room triggers CANNOT be in any prefab! They must be in the base map file. This is because the sound converter doesn't check prefabs when looking for ambient data in your maps.

Scobalula said:

The ambient data is compiled directly into the sound xasset struct, which is why snd_convert.exe needs to know the map file; snd_convert will generate a file called <MapName>.ambientgeometry.json for the linker to use.
